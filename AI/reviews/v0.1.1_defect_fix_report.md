# flatten-polymorph v0.1.1 Defect Fix Report

## Executive Summary

Fixed the v0.1.1 release-blocking and safety defects without weakening safety policy. The planner now rejects `PROBABLY_CLOSED`, guarded rewrites fall back to original dynamic dispatch for unmatched receiver types, trace binding requires method-name agreement, and the release gate validates the built wheel in a clean venv.

## Defect Summary

- P0-1 `flatten_polymorph.__main__`: verified complete module guard with `raise SystemExit(main())`.
  - Repro command: `python -m flatten_polymorph --help`
  - Test: `tests/test_v011_regressions.py::test_flatten_polymorph_module_help_matches_flatten_help`
- P0-2 CLI annotation: source and installed package strict mypy pass.
  - Repro command: `python -m mypy --strict src\flatten src\flatten_polymorph`
  - Log: `AI/logs/v011_mypy_strict_final.log`
- P0-3 release gate: added `scripts/release_gate.ps1` and CI `release-gate`.
  - Repro command: `powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\release_gate.ps1`
  - Log: `AI/logs/v011_release_gate_green.log`
- P1-1 single-file local completeness is no longer CLOSED.
  - Root cause: local hierarchy completeness could be treated as rewrite-safe evidence.
  - Fix: return `PROBABLY_CLOSED` without allowing rewrite unless positive CLOSED evidence exists.
  - Test: `tests/test_v011_regressions.py::test_single_file_static_analysis_does_not_close_public_hierarchy`
- P1-2 guarded chain now includes dynamic fallback.
  - Root cause: last else branch called an arbitrary observed implementation.
  - Fix: multiple-receiver replacement ends with `receiver.method(...)`; temp receiver paths use `_flatten_receiver_N.method(...)`.
  - Test: `test_guarded_rewrite_falls_back_to_original_dynamic_dispatch_for_unmatched_type`
- P1-3 trace binding requires method-name match and reports unbound observations.
  - Root cause: line/column matching could bind the wrong method or fail nested calls silently.
  - Fix: candidate matching includes `method_name`; trace prints bound/unbound summary on unbound evidence.
  - Tests: `test_trace_binding_requires_matching_method_name_on_same_line`, `test_nested_call_trace_binds_inner_method`
- P1-4 dispatch order is no longer string reverse order.
  - Root cause: receiver type names were sorted lexically.
  - Fix: order by restored class MRO depth, then deterministic class identity.
  - Test: `test_planner_orders_subclasses_before_parents_in_guard_chain`
- P2-1 tracer value capture is opt-in via `--capture-values`.
  - Test: `test_tracer_default_does_not_call_deepcopy_for_cli_minimal_capture`
- P2-2 removed unreachable collapse transformer branches and preserved plan fields via `dataclasses.replace`.
  - Test: `tests/test_planner_field_preservation.py`
- P2-3 confidence is documented and implemented as a CLOSED-only ordinal score.
  - Test: `tests/test_confidence_contract.py`
- P2-4 harness stdout/stderr capture uses `contextlib.redirect_stdout/redirect_stderr`; `_jsonable` handles cycles.
  - Test: `test_jsonable_handles_cycles_without_recursion_error`
